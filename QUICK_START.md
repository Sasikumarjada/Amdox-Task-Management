# üöÄ Quick Start Guide - Amdox Task Management

## 5-Minute Setup

### Step 1: Prerequisites Check
```bash
# Check Java version (needs 17+)
java -version

# Check Maven
mvn -version

# Check MySQL
mysql --version
```

### Step 2: Database Setup
```sql
-- Login to MySQL
mysql -u root -p

-- Create database
CREATE DATABASE amdox_task_db;

-- Exit MySQL
exit;
```

### Step 3: Configure Application
Edit `src/main/resources/application.properties`:
```properties
# Update these lines
spring.datasource.username=root
spring.datasource.password=YOUR_MYSQL_PASSWORD
```

### Step 4: Run Application
```bash
# Build
mvn clean install -DskipTests

# Run
mvn spring-boot:run
```

‚úÖ Application should start on **http://localhost:8080**

## üìã Test the API

### 1. Register a User
```bash
curl -X POST http://localhost:8080/api/auth/register \
  -H "Content-Type: application/json" \
  -d '{
    "username": "admin",
    "email": "admin@amdox.com",
    "password": "admin123",
    "fullName": "Admin User"
  }'
```

**Response:** You'll get a JWT token
```json
{
  "token": "eyJhbGciOiJIUzI1NiIsInR5cCI6IkpXVCJ9...",
  "username": "admin",
  "email": "admin@amdox.com",
  "fullName": "Admin User",
  "role": "VIEWER"
}
```

### 2. Update User to ADMIN Role
First, login to MySQL and update:
```sql
USE amdox_task_db;
UPDATE users SET role = 'ADMIN' WHERE username = 'admin';
```

### 3. Create a Task
```bash
curl -X POST http://localhost:8080/api/tasks \
  -H "Content-Type: application/json" \
  -H "Authorization: Bearer YOUR_JWT_TOKEN" \
  -d '{
    "title": "Setup Development Environment",
    "description": "Install all required tools and configure the workspace",
    "priority": "HIGH",
    "deadline": "2024-12-31T23:59:59",
    "category": "Development",
    "tags": "setup,environment"
  }'
```

### 4. Get All Tasks
```bash
curl -X GET http://localhost:8080/api/tasks \
  -H "Authorization: Bearer YOUR_JWT_TOKEN"
```

## üêõ Troubleshooting

### Issue: Port 8080 already in use
**Solution:**
```bash
# Find process
lsof -i :8080

# Kill process
kill -9 <PID>

# Or change port in application.properties
server.port=8081
```

### Issue: Cannot connect to database
**Solution:**
1. Check MySQL is running: `sudo service mysql status`
2. Verify credentials in `application.properties`
3. Ensure database exists: `SHOW DATABASES;`

### Issue: JWT token expired
**Solution:**
- Login again to get a new token
- Tokens expire after 24 hours by default
